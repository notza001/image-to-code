# Lesson 01: Evidence-Based Design

**Date**: 2026-02-03
**Context**: สร้างปุ่ม Siam Rent แล้วผิดหลายครั้ง

---

## หลักการ

> **"อย่าคิดไปเอง หลักฐานมันชัด ให้เอาหลักฐานเป็นหลัก"**

---

## สิ่งที่ทำผิด

| ผิด | เพราะ |
|-----|-------|
| ใส่มุมเหลืองที่ไม่มี | คิดไปเองว่า "น่าจะมี" |
| Secondary text สีแดง | เดาเอา |
| Secondary Hover ขอบเทา | ไม่ได้ดูรูปดีๆ |

---

## วิธีที่ถูก

```
1. ดูรูป
2. เห็นอะไร → บันทึกแค่นั้น
3. ไม่เพิ่มสิ่งที่ไม่มี
4. ไม่เดา
5. ถ้าไม่แน่ใจ → ถาม
```

---

## Checklist ก่อนเขียนโค้ด

- [ ] ดูรูปแล้ว 3 รอบ
- [ ] ลิสต์ทุก element ที่เห็น
- [ ] ไม่มีอะไรที่ "คิดว่าน่าจะมี"
- [ ] ทุกอย่างมาจากรูปเท่านั้น
- [ ] ถ้ารูปเล็ก/ไม่ชัด → ขอรูปขยายก่อน

---

## Lesson 1.1: รูปเล็ก = เห็นสีผิด

**ปัญหา:** รูปเล็กเกินไป ทำให้ Claude Vision เห็นสีผิด

**ตัวอย่าง:**
- รูปเล็ก → เห็น Secondary text เป็นสีแดง
- รูปขยาย → เห็นชัดว่าเป็นสีขาว

**วิธีแก้:**
- ถ้าไม่แน่ใจสี → ขอรูปขยายเฉพาะส่วนนั้น
- หรือ crop ส่วนที่ต้องการดูให้ใหญ่ขึ้น

---

## สรุป

**Evidence > Assumption**

ดูหลักฐาน ไม่ใช่คิดเอง

**ถ้าไม่ชัด → ขอรูปขยาย**
